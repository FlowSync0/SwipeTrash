## Product Facts

Name: SwipeTrash

Tagline options:
- Clean old files in 30 seconds a day.
- Swipe through old downloads. Keep or trash.
- A tiny daily cleanup ritual for your computer.

Short description:
SwipeTrash is a free desktop app for macOS, Windows, and Linux that shows old files from Downloads and Documents as swipe cards. Swipe right to keep, swipe left to move junk to the system trash.

Long description:
SwipeTrash turns file cleanup into a small daily habit. Instead of opening a giant folder and deciding everything at once, you review a few old files with a large preview, then either keep them or move them to the system trash. It avoids hidden files, app folders, game/mod assets, dependency folders, caches, source/build folders, keys, databases, and sensitive names by default.

## Product Hunt

Tagline:
Clean old files in 30 seconds a day.

Description:
SwipeTrash shows old Downloads and Documents files as swipe cards. Keep what matters, move junk to the system trash, and avoid the giant "I need to clean my computer" session.

First maker comment:
I built SwipeTrash because my Downloads folder kept turning into a pile of screenshots, archives, installers, and exports that I only needed once.

Most cleaner apps feel too aggressive, and manual cleanup becomes a big session that I postpone. SwipeTrash makes it intentionally small: open it, review a few files, swipe right to keep or left to move to the system trash.

The safety model is conservative: it skips hidden files, app folders, game/mod assets, dependency folders, caches, source/build folders, keys, databases, and sensitive names. It also moves files to the system trash instead of deleting them permanently.

It is open source and runs on macOS, Windows, and Linux. I would especially like feedback on the file filtering rules and the swipe interaction.

## Hacker News

Body:
I built SwipeTrash, a small open-source desktop app for cleaning old Downloads and Documents files in short daily sessions.

The idea is to avoid the big "I need to clean my computer" cleanup session. The app shows one old file at a time with a large preview. Swipe right to keep it, swipe left to move it to the system trash.

Safety was the main design constraint. It skips hidden files, application folders, game/mod assets, dependency folders, caches, source/build folders, keys, databases, and sensitive names. It does not permanently delete files.

It is built with Electron, React, and TypeScript, and the release ships macOS, Windows, and Linux builds from GitHub Actions.

I would appreciate feedback on the filtering rules and whether the interaction feels trustworthy enough for a file cleanup app.
